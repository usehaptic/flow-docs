Secrets are sensitive values such as provider keys, tokens, and credentials.

## Storage model

Secret metadata is stored in Postgres. Secret values are stored in HCP Vault KV v2 through `flow-vault-service`.

## Write-only behavior

Browser-facing list responses do not return secret values. The UI only shows metadata and a write-only badge.

## Supported scopes

- `account`: reusable account-level secrets.
- `project`: secrets that belong to a specific project.

## Environments

Project secrets currently use:

- `Development`
- `Production`

## Duplicate handling

Flow prevents duplicate secret names for the same project, environment, and item type.

## Deleting secrets

Deleting a secret requires confirmation. Deletion removes both Vault value and metadata.
