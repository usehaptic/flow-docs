# Contribute to Flow Docs

Flow Docs documents Flow Cloud for developers and internal contributors.

## Local development

1. Clone this repository.
2. Install the Mintlify CLI: `npm i -g mint`.
3. Run `mint dev` from the repository root.
4. Preview changes at `http://localhost:3000`.
5. Run `mint broken-links` before opening a pull request.

## Writing guidelines

- Use active voice.
- Address the reader directly.
- Keep sentences concise.
- Use Flow terminology consistently.
- Mark planned features clearly.
- Avoid exposing credentials, tokens, private URLs, or customer data.

## What to document

- Product workflows that developers use.
- Current service responsibilities and API contracts.
- Operational setup and deployment notes.
- Known implementation boundaries and planned behavior.
