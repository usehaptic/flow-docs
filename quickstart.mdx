This guide walks through the primary Flow Cloud workflow from project creation to project configuration.

<Steps>
  <Step title="Create or open a project">
    Start from the projects dashboard. Create a new project or open an existing one to load the Flow IDE.
  </Step>
  <Step title="Edit files in the IDE">
    Use the **Code** workspace to open files from the project file tree. Flow loads file content from the project service and saves changes back to the virtual file system.
  </Step>
  <Step title="Configure project values">
    Open **Project settings** from the workspace menu. Add secrets, environment variables, signing assets, and capabilities as needed.
  </Step>
  <Step title="Run a build">
    Click **Run**. Flow sends project files plus project context headers to the build endpoint and streams build output back into the IDE.
  </Step>
  <Step title="Use Flow AI">
    Click **Flow AI** to open the assistant rail. Flow AI can inspect project context and write generated code directly into files.
  </Step>
</Steps>

## Before you begin

- You need a Flow account.
- You need an active project.
- If you want Flow AI to call a model provider, connect provider API keys from account settings.
- If your app needs secrets at build time, add them in Project Settings before running a build.
